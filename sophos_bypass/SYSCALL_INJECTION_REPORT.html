💡 Key Insight: AV products hook DLL functions, not the syscall instruction itself. By calling syscall directly with the correct system call number, we bypass ALL userland hooks while achieving the same kernel functionality.

🔢 System Call Mechanism

System calls are the interface between userland and the Windows kernel. Each syscall has a unique number that varies between Windows versions.

Syscall Numbers (Windows 10 21H2)

Function Syscall Number Purpose
NtAllocateVirtualMemory 0x18 Allocate memory in target process
NtWriteVirtualMemory 0x3A Write shellcode to allocated memory
NtProtectVirtualMemory 0x50 Change memory permissions to RWX
NtCreateThreadEx 0xC1 Create remote thread in target

Syscall Assembly Structure

; Standard NTDLL syscall stub pattern
mov   r10, rcx          ; Move first param to r10 (Windows x64 convention)
mov   eax, 0x18         ; Syscall number for NtAllocateVirtualMemory
syscall                  ; Transition to kernel mode
ret                      ; Return to caller
⚠️ Version Dependency: Syscall numbers change between Windows versions. Production tools should dynamically resolve these by parsing ntdll.dll export table or using techniques like "Hell's Gate" for runtime resolution.

💉 Process Injection Technique

Our injection uses direct syscalls to perform classic shellcode injection into explorer.exe, a trusted system process that blends with normal system activity.

Injection Steps

  1. Open Target Process - Get handle to explorer.exe (PID: dynamic)
  2. Allocate Memory - Reserve RW memory in target using NtAllocateVirtualMemory
  3. Write Shellcode - Copy Meterpreter payload using NtWriteVirtualMemory
  4. Change Permissions - Set memory to RX using NtProtectVirtualMemory
  5. Execute - Create remote thread using NtCreateThreadEx

Target Process Selection

✅ Good Targets
